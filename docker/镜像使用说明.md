# BindCraft 三镜像使用说明(单环境分阶段)

> 面向"拉取已构建镜像、分别测试每个环境功能"的读者。讲清楚三个镜像各是什么、各自的输入输出,以及如何把上一阶段的产物交给下一环境,**逐阶段单独跑通**。

---

## 1. 这是什么

BindCraft 是从头设计蛋白 binder 的流水线(AlphaFold2 反向传播 + ProteinMPNN + PyRosetta 打分)。本套 Docker 把依赖栈拆成**三个独立镜像**,每个可单独构建、单独运行:

| 镜像 tag | 角色 | 装了什么 | 要 GPU? |
|---|---|---|:---:|
| `ai4science/bindcraft:gpu` | AF2/colabdesign 计算 | jax + colabdesign + AF2 权重(**无 pyrosetta**) | 是 |
| `ai4science/bindcraft:rosetta` | PyRosetta 打分 | PyRosetta + DAlphaBall + dssp(**无 jax**) | 否 |
| `ai4science/bindcraft:orchestrator` | 入队 / 聚合(bookkeeping) | 仅 pandas(**无 jax、无 pyrosetta**) | 否 |

拆分靠环境变量 `BINDCRAFT_MODE`(`local`/`gpu`/`rosetta`)切换依赖导入。本文只覆盖**单环境分阶段**用法。

---

## 2. 拉取镜像

DockerHub 不通时用镜像站 `docker.1ms.run`:

```bash
docker pull docker.1ms.run/ai4science/bindcraft:rosetta
docker pull docker.1ms.run/ai4science/bindcraft:gpu          # 大(约 15GB,含 5.3GB 权重),慢
docker pull docker.1ms.run/ai4science/bindcraft:orchestrator
```

> 镜像站对大层偶发 `unknown blob` / `unexpected EOF`,失败重试即可(已下载层会续传):
> ```bash
> for i in $(seq 1 8); do docker pull docker.1ms.run/ai4science/bindcraft:gpu && break; sleep 10; done
> ```

> ⚠️ 这些镜像基于 micromamba,**`python` 只有走默认 `_entrypoint.sh` 激活后才在 PATH 里**。单独跑自定义命令时,要么把命令直接当参数传(orchestrator 是普通 python 镜像,可直接 `python`),要么对 gpu/rosetta 用 `--entrypoint /usr/local/bin/_entrypoint.sh`。

## 3. 三个镜像各自的输入 / 输出

各镜像通过**共享目录**交接数据:把一个宿主目录挂到每个容器的 `/workspace`,上一阶段写进去的文件,下一阶段直接读。**只交接文件路径,不走网络。**

### 3.1 `orchestrator` —— 入队 / 聚合

普通 `python:3.10-slim`,ENTRYPOINT 即 `python -u /app/orchestrator.py`,靠环境变量控制行为。

- **入队**(`ENQUEUE=1 WAIT=0`):
  - 输入:`${BATCH_IN}/*.json`(target 设置,schema 同 `settings_target/*.json`)
  - 输出:复制到 `${QUEUE}/pending/`,每个 job JSON 携带 `design_path`
- **聚合**(`ENQUEUE=0 WAIT=1`):
  - 输入:已落定到 `${QUEUE}/done/` 的 job + 各 `design_path/final_design_stats.csv`
  - 输出:`${WORKSPACE}/combined_final_stats.csv`(每行前缀 target 名)

### 3.2 `gpu` —— AF2 / colabdesign 计算

跑 `bindcraft.py` 的设计循环(hallucinate / MPNN / 重预测)。AF2 权重已打包在 `/app/params`,运行时不下载。

- **输入**:target PDB + 设置(hotspot、binder 长度、seed 等)
- **输出**:trajectory / 预测的 PDB 结构(写入 `/workspace`)
- **注意**:`bindcraft.py` 在 `BINDCRAFT_MODE=gpu` 下会把 relax/score 转发到 rosetta 服务;**单独测 gpu 功能**时,直接调 colabdesign 产出 PDB 即可(见 §4 Stage B),不依赖 rosetta。

### 3.3 `rosetta` —— PyRosetta 打分

提供 BindCraft 用到的 PyRosetta 操作。单独使用时直接 import 调用即可。

| 函数 | 输入 | 输出 |
|---|---|---|
| `pr_relax(pdb_file, relaxed_pdb_path)` | 复合物 PDB | 写出 relax 后 PDB |
| `score_interface(pdb_file, binder_chain)` | 复合物 PDB | `scores`(含 `interface_dG`、`interface_nres` 等)、`interface_AA`、`interface_residues` |
| `unaligned_rmsd(...)` / `align_pdbs(...)` | 两个 PDB + 链 ID | rmsd / 对齐写回 |

> `score_interface` 会 exec 外部二进制 `DAlphaBall.gcc`(算 BuriedUnsatHbonds),它动态链接 `libgfortran.so.5`。镜像已设 `LD_LIBRARY_PATH=/opt/conda/lib` 让其可找到;否则会报 `libgfortran.so.5: cannot open shared object file`。

## 4. 分阶段功能测试(逐环境跑通,产物链式交接)

用一个共享宿主目录挂到各容器 `/workspace`,每阶段一个一次性容器,上一阶段产物喂给下一环境。下面用内置示例 `docker/examples/PDL1_smoke.json` 与 `example/PDL1.pdb` 演示。

```bash
cd /path/to/BindCraft
IMG=docker.1ms.run/ai4science/bindcraft
HOST=$PWD/.stagetest && rm -rf "$HOST" && mkdir -p "$HOST"
mkdir -p batch_inputs && cp docker/examples/PDL1_smoke.json batch_inputs/
```

### Stage A — orchestrator 入队(无 GPU)

```bash
docker run --rm -v "$HOST":/workspace -v "$PWD/batch_inputs":/batch_inputs:ro \
  -e BATCH_IN=/batch_inputs -e QUEUE=/workspace/queue -e WORKSPACE=/workspace \
  -e ENQUEUE=1 -e WAIT=0 \
  $IMG:orchestrator
# 期望:enqueued PDL1_smoke.json;产物 $HOST/queue/pending/PDL1_smoke.json
```

### Stage B — gpu 折叠产出复合物 PDB(需 GPU)

把下面脚本存为 `/tmp/stage_gpu.py`:

```python
import sys
from colabdesign import mk_afdesign_model, clear_mem
clear_mem()
m = mk_afdesign_model(protocol="binder", data_dir="/app", use_multimer=True,
                      num_recycles=1, best_metric="loss")
m.prep_inputs(pdb_filename="/app/example/PDL1.pdb", chain="A",
              binder_len=70, hotspot="56", seed=0)
m.design_logits(iters=3, num_models=1, save_best=True)   # 冒烟用 3 轮;真实 50+
m.save_pdb("/workspace/stage_gpu_complex.pdb")
print("WROTE /workspace/stage_gpu_complex.pdb")
```

```bash
docker run --rm --gpus '"device=0"' \
  -v "$HOST":/workspace -v /tmp/stage_gpu.py:/stage_gpu.py:ro \
  --entrypoint /usr/local/bin/_entrypoint.sh \
  $IMG:gpu python -u /stage_gpu.py
# 期望:loss 逐轮下降,输出含 A/B 双链的 stage_gpu_complex.pdb
```

### Stage C — rosetta 消费 gpu 的 PDB,relax + score(无 GPU)

把下面脚本存为 `/tmp/stage_rosetta.py`:

```python
import os, csv
import pyrosetta as pr
pr.init('-ignore_unrecognized_res -ignore_zero_occupancy -mute all '
        '-holes:dalphaball /app/functions/DAlphaBall.gcc -corrections::beta_nov16 true -relax:default_repeats 1')
from functions.pyrosetta_utils import pr_relax, score_interface
pr_relax("/workspace/stage_gpu_complex.pdb", "/workspace/stage_relaxed.pdb")
scores, _, _ = score_interface("/workspace/stage_relaxed.pdb", "B")
os.makedirs("/workspace/designs/PDL1_smoke", exist_ok=True)
with open("/workspace/designs/PDL1_smoke/final_design_stats.csv", "w", newline="") as fh:
    w = csv.writer(fh); w.writerow(["Design", "interface_dG", "interface_nres"])
    w.writerow(["PDL1smoke_stage", scores.get("interface_dG"), scores.get("interface_nres")])
print("interface_dG=", scores.get("interface_dG"), "nres=", scores.get("interface_nres"))
```

```bash
docker run --rm -v "$HOST":/workspace -v /tmp/stage_rosetta.py:/stage_rosetta.py:ro \
  -e BINDCRAFT_MODE=rosetta \
  --entrypoint /usr/local/bin/_entrypoint.sh \
  $IMG:rosetta python -u /stage_rosetta.py
# 期望:打印 interface_dG/nres;产物 stage_relaxed.pdb + designs/PDL1_smoke/final_design_stats.csv
```

### Stage D — orchestrator 聚合(无 GPU)

```bash
# 模拟该 job 已完成:pending -> done
docker run --rm -v "$HOST":/w alpine sh -c "mkdir -p /w/queue/done && mv /w/queue/pending/PDL1_smoke.json /w/queue/done/"

docker run --rm -v "$HOST":/workspace \
  -e QUEUE=/workspace/queue -e WORKSPACE=/workspace -e ENQUEUE=0 -e WAIT=1 -e POLL=2 \
  $IMG:orchestrator
# 期望:wrote combined_final_stats.csv(1 rows from 1 targets)
docker run --rm -v "$HOST":/w alpine cat /w/combined_final_stats.csv
```

最终输出形如:

```
target,Design,interface_dG,interface_nres
PDL1_smoke.json,PDL1smoke_stage,-28.9,14
```

### 清理

```bash
docker run --rm -v "$PWD":/p alpine rm -rf /p/.stagetest    # 容器写的是 root,用容器删
rm -f /tmp/stage_gpu.py /tmp/stage_rosetta.py
```

## 5. 单镜像快速自检(无 GPU)

确认三个镜像各自只装了该装的栈:

```bash
IMG=docker.1ms.run/ai4science/bindcraft
docker run --rm $IMG:gpu          python -c "import jax, colabdesign; print('gpu ok, 无需 pyrosetta')"
docker run --rm $IMG:rosetta      python -c "import pyrosetta; print('rosetta ok, 无需 jax')"
docker run --rm $IMG:orchestrator python -c "import pandas; print('orchestrator ok')"
```

GPU 可见性(需 nvidia-container-toolkit):

```bash
docker run --rm --gpus all $IMG:gpu python -c "import jax; print(jax.devices())"
```

## 6. 常见问题排查

| 现象 | 排查 |
|---|---|
| `python: executable file not found` | micromamba 镜像下别用 `--entrypoint python`;用 `--entrypoint /usr/local/bin/_entrypoint.sh` 再接 `python ...`(orchestrator 例外,可直接 `python`) |
| `score_interface` 报 `libgfortran.so.5: cannot open shared object file` | rosetta 镜像需 `LD_LIBRARY_PATH=/opt/conda/lib`(新镜像已内置;旧镜像运行时加 `-e LD_LIBRARY_PATH=/opt/conda/lib`) |
| gpu 阶段找不到 AF2 权重 | `data_dir` 应传 `/app`(权重在 `/app/params/*.npz`) |
| 共享卷里文件读不到 | 三阶段必须挂同一宿主目录到相同路径 `/workspace`;`design_path` 要在 `/workspace` 下 |
| 镜像拉取 `unknown blob` / EOF | 镜像站大层不稳,重试(已下载层续传) |
| GPU 设备列表为空 | 宿主装 NVIDIA 驱动 + nvidia-container-toolkit |
